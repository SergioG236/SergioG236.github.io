Sergio Gerardo Flores Infante

Resumen

En el siguiente documento se presentará el cómo se realizó la implementación del sistema de administración, control y repositorio de la plataforma CAPASITI las herramientas utilizadas tales como un servidor local para la realización de pruebas antes de las modificaciones en línea y un editor de texto para la modificación del código, terminando con la correcta implementación del sistema CAPASITI. En este reporte se mostrará lo realizado en la semana que comprende los días martes 19 de noviembre de 2019 hasta viernes 22 de noviembre de 2019 en las instalaciones de la SUBSECRETARIA DE INNOVACIÓN Y TECNOLOGÍAS DE LA INFORMACIÓN .

Abstract

The following document will present how the implementation of the administration, control and repository system of the CAPASITI platform was performed, the tools used such as a local server for testing before online modifications and a text editor for the code modification, ending with the correct implementation of the CAPASITI system. This report will show what was done in the week that includes the days Tuesday, November 19, 2019 until Friday, November 22, 2019 at the facilities of the “SUBSECRETARIA DE INNOVACIÓN Y TECNOLOGÍAS DE LA INFORMACIÓN”.

Introducción

Hoy día dentro de cualquier empresa, del tipo que sea, es necesario contar con un equipo que se haga cargo de las actividades que corresponden al área de las tecnologías, es decir, páginas web, redes, entre otras. Esto con el fin de poder tener comunicación dentro de la misma, tener un lugar donde mantener seguros documentos importantes o en casos más extremos, asegurarse de que usuarios no deseados accedan a donde les plazca, por dar algunos ejemplos. Dentro de la Subsecretaria de Innovación y Tecnologías de la Información varias de estas actividades son llevadas a cabo por sus especialistas, por ello a manera de consideración se permitió el desarrollo de las estancias a los alumnos que cursan la carrera de ingeniería en tecnologías de la información provenientes de la Universidad Politécnica de Victoria, esto con el fin de mostrar cómo es que se desarrollan estas actividades dentro de un área empresarial real. El fin de la estancia 1 es dar a los alumnos los conocimientos y herramientas que se necesitaran una vez se dé por concluida la carrera que se encuentran cursando, haciendo que en un futuro sea más sencillo poder desenvolverse dentro de estos sectores y puedan adquirir una mentalidad más madura en cuanto a temas de trabajo se refiere.

Tecnologías

Es aquella rama de la tecnología cuyo principal nicho se encuentra dentro de lo referente a las telecomunicaciones, equipos electrónicos y programación. Se hace uso de la capacidad que tienen las máquinas para poder realizar tareas que comúnmente los humanos no son capaces de llevar a cabo, siendo especialmente influenciado en los ámbitos de las matemáticas y la lógica.

Herramientas utilizadas.

Para poder llevar a cabo las actividades requerías a lo largo del periodo de estancias, es necesario contar con el equipo adecuado con el cuál poder desarrollar cada una de las actividades. Dentro de estas se puede encontrar softwares y frameworks como los descritos a continuación.

MySQL

Debido a la implementación de una base de datos se optó por el uso de uno de los mejores sistemas de gestión que se tiene hoy día como lo es MySQL. MySQL es un software creado por la empresa sueca MySQL AB por David Axmark, Allan Larsson y Michael Widenius. Su lenguaje de programación se trata de Structured Query Lenguaje (SQL) desarrollado en 1981 por IBM. Como muchos otros sistemas de gestión MySQL cuenta con elementos que apoyan al mantenimiento y uso de las bases de datos, tales como datos relacionales, integridad referencial y transacciones.

CakePHP

CakePHP es un framework para PHP, creado en 2005 por Michal Tatarynowicz. En sus comienzos se comenzó buscando ser una versión del framework Rapid Application. Al igual que otros framework del mismo estilo, CakePHP proporciona una mejor comunicación con la base de datos. Además, ofrece las herramientas necesarias para el desarrollo dependiendo de las necesidades del usuario.

WAMPP

Se trata de un servidor cuyas principales características son su funcionalidad con la base de datos de MySQL, el uso de apache, y su uso con el lenguaje PHP, debido a los puntos anteriores y a su fácil implementación fue elegido para poder desarrollar la practica requerida.

FileZilla

Es una aplicación desarrollada en 2001 por Tim Koose, tiene la funcionalidad de descargar páginas web de su respectivo servidor, siempre y cuando se cuente con la información de esta misma. Para el desarrollo de las estancias fue indispensable debido a que se requería descargar todos los paquetes de un servidor a partir del cual se realizarían cambios y se vería como es su comportamiento.

Desarrollo

Primeros pasos

El primer paso para comenzar con la práctica, fue facilitada una carpeta con la cuál acceder de manera local a la página que se deseaba modificar, esta fue ingresada como se mostró anteriormente a la carpeta www de wamp, sin embargo, se encontró que al momento de querer acceder de manera local se redirigía de manera automática a otra red.
Después de esto, haciendo uso de FileZilla, se descargaron los archivos que se encontraban en el servidor. Pero antes de esto se necesitó conectar a través de la red IP del usuario para que se le cedieran los permisos para poder editar el servidor capasiti.tamaulipas.gob.mx pasando por primera instancia conectarse a la IP de la Torre Bicentenario para poder acceder a dicho servidor.

Antes de subir la página

Como otra de las tareas encomendadas, fue necesario realizar un re direccionamiento, pero ahora en lugar de acceder al inicio se buscaba acceder a la sección de cursos que se encuentran dentro de CAPASITI. Para poder realizar esto se accedió a app->controler->components->appcontroler líneas 156-157 y se cambió la dirección.
Una de las ultimas tareas antes de poder subirlo al servidor fue cambiar las opciones de Login y Logout, es decir, las opciones que redirigen a una u otra parte de la página al ingresar un usuario, esto se encuentra en app->components->appController en $this->Auth-> loginAction = '/usuarios/login/'; logoutRedirect = '/usuarios/login/'; loginRedirect = '/pages/index/'.

Subiendo la página

Con la plataforma en funcionamiento, se comenzó con la depuración de la base de datos, esto gracias a los permisos de administrador que se consiguieron, siendo más fácil la eliminación de elementos no necesarios, como lo eran algunos usuarios y cursos que se encontraban, dejando solo un grupo de usuarios que funcionan como pruebas para revisar si cada uno tiene sus permisos correspondientes.
Se encontró que el error surgía debido a que en el código ubicado en UsuariosCalendarizadosController.php en las líneas 591 a la 595 se hacía mal uso de una palabra reservada, además de que había palabras mal escritas que hacían referencia a tablas de la base de datos, por lo cual nunca se podía acceder a la misma y generar los reportes.

Para terminar

Ahora, con las nuevas actualizaciones, subidas utilizando nuevamente FileZilla, se realizaron algunas pruebas con personas ajenas al proyecto, esto con el fin de observar que tan didáctica es la página y lograr conseguir cualquier otro error que pueda surgir y que el equipo no haya encontrado con anterioridad.
Los errores encontrados fueron solucionados y se termino con los cambios realizados, haciendo entrega del trabajo en su totalidad
